# Security Policy

Pluma is a local-first desktop Markdown editor. Security reports are especially
important when they involve local file access, Electron IPC or preload
boundaries, Markdown rendering, export behavior, or release packaging.

## Reporting A Vulnerability

Please do not file public issues for vulnerabilities.

Use GitHub private vulnerability reporting when it is available for this
repository. If it is not available, open a brief public issue asking for a
private contact channel without including vulnerability details.

Helpful reports include:

- Affected commit or release.
- Operating system and architecture.
- Clear reproduction steps.
- Expected and actual impact.
- Whether local files, rendered Markdown, exported HTML, IPC, or packaged
  releases are involved.

## Supported Versions

Pluma is currently alpha-stage software. Security fixes are prioritized on the
main branch and the latest published alpha release when releases are available.
